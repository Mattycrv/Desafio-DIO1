O que são Clients e Servers

Client:

Um cliente é um programa que, como parte de sua operação, depende do envio de uma solicitação a outro programa ou a um hardware ou software de computador que acessa um serviço disponibilizado por um servidor (que pode ou não estar localizado em outro computador). Por exemplo, navegadores da web são clientes que se conectam a servidores da web e recuperam páginas da web para exibição. Clientes de e-mail recuperam e-mails de servidores de e-mail . O bate-papo online usa uma variedade de clientes, que variam no protocolo de bate-papo usado. Jogos de vídeo multijogador ou jogos de vídeo online podem ser executados como um cliente em cada computador. O termo "cliente" também pode ser aplicado a computadores ou dispositivos que executam o software cliente ou usuários que usam o software cliente.


Server:

Em programação, "server" refere-se a um computador ou software que fornece serviços ou recursos para outros computadores, conhecidos como clientes, em uma rede. A programação do lado do servidor (server-side) é a lógica que roda no servidor, processando requisições e gerando respostas, muitas vezes interagindo com bancos de dados e garantindo a segurança da aplicação.


Server-Side e Client-Side

Server-side refere-se a operações realizadas no servidor em um contexto cliente-servidor, ou seja, não no navegador do usuário. É o oposto de "client-side", que descreve operações realizadas no dispositivo do usuário. O lado do servidor é frequentemente chamado de "back-end" e inclui a lógica principal da aplicação, como processamento de dados, interação com bancos de dados e autenticação.


O que acontece no lado do servidor?

Processamento de dados:

O servidor recebe solicitações do cliente (navegador) e as processa, realizando cálculos, validações e outras operações lógicas.


Acesso a bancos de dados:

O servidor se comunica com bancos de dados para armazenar, recuperar e atualizar informações.


Autenticação e segurança:

O servidor lida com a autenticação de usuários, verificando se eles têm permissão para acessar determinados recursos.


Geração de conteúdo dinâmico:

O servidor pode gerar conteúdo HTML personalizado para cada usuário, com base em dados armazenados ou informações da sessão.


Renderização do lado do servidor (SSR):

O servidor gera o HTML completo da página antes de enviá-lo ao cliente, o que é benéfico para SEO e carregamento inicial rápido.


Estrutura Básica do HTML

A estrutura básica do HTML consiste em três elementos principais: DOCTYPE, html, html, head e body. A declaração DOCTYPE html informa ao navegador que o documento é HTML5. O elementohtml envolve todo o conteúdo da página, enquanto head contém metadados e links para arquivos externos, e body contém o conteúdo visível da página
